### 2C. AI 能耗预测引擎（必须完成）

> 与 LLM 故障诊断（2A）互补：2A 解决"已发生故障→怎么修"，2C 解决"未来能耗→怎么省"

#### 数据准备与特征工程
- [ ] `data/export_training_data.py` — 从 TimescaleDB 导出传感器历史数据为 CSV
- [ ] 数据清洗：去异常值（3σ 法）、线性插值补全缺失、标注故障时段和电价时段
- [ ] 特征工程管道（`ai_agents/features/feature_pipeline.py`）：
  - 时间特征：NSM 编码、小时 sin/cos、星期几、是否节假日
  - 滞后特征：t-1, t-2, t-3, t-6, t-12 的功率/温度/振动值
  - 滚动窗口特征：5min/15min/30min 均值、标准差、最大最小值
  - 差分特征：功率/温度的一阶/二阶差分
  - 电价特征：当前电价时段 one-hot 编码
  - 工况特征：operatingStatus one-hot 编码
- [ ] 数据集划分：训练 70% / 验证 15% / 测试 15%（按时间顺序，不打乱）

#### 模型搭建与训练
- [ ] `ai_agents/models/lstm_model.py` — LSTM 模型（输入 30min 窗口 → 输出未来 30min 预测，MSE/MAE 损失）
- [ ] `ai_agents/models/transformer_model.py` — Transformer Encoder 时序预测（位置编码 + 多头注意力）
- [ ] `ai_agents/models/train.py` — 训练脚本（超参数搜索、Early Stopping、Model Checkpoint、TensorBoard）
- [ ] 评估指标：MAPE < 8%、RMSE、MAE

## 附录 D：系统预期功能描述（面向客户）

> 以下是对智驭能效系统功能的通俗描述，适用于向非技术背景的客户/评委介绍系统价值。

**智驭能效**是一套面向钢铁厂的智慧能源管理系统。它像一个"24 小时不休息的能源管家"，帮工厂解决三个核心问题：**省电费、找故障、管维修**。

**实时监控，一目了然。** 系统把全厂核心设备（电弧炉、水泵、空压机）的运行状态实时显示在一块大屏上。哪个设备在运行、哪个设备温度偏高、哪个设备离线了，一眼就能看到。总控室的值班人员不用去现场巡检，坐在屏幕前就能掌握全厂动态。

**聪明告警，出事早知道。** 传统的报警系统经常"狼来了"——动不动就响，分不清是真故障还是正常波动。智驭能效不会这样。它结合多条传感器数据交叉判断（比如温度高 + 压力低 + 设备在高速运转 = 冷却系统可能出了大问题），只有真正危险的时候才报警。更厉害的是，报警的同时，系统会调用 AI 大模型自动分析故障原因，生成详细的维修方案——就像有位经验丰富的老工程师在旁边指导一样。

**AI 帮你省电费。** 电力有峰谷电价——晚上便宜、白天贵、傍晚最贵。系统会预测未来半小时的用电量，结合电价走势，自动给出排产建议。比如："建议这台电炉推迟 30 分钟开工，等过了尖峰电价再启动，这样可以省 800 多块钱。"调度员点一下确认就可以执行。

**维修管理不再乱。** 设备出故障后，系统自动生成维修工单，根据故障类型匹配合适的维修师傅，连带把 AI 分析的故障原因、维修步骤、需要什么备件、注意什么安全问题，一起推送给师傅的手机/电脑。师傅照着步骤修就行，修好了在系统里点"完成"，整个流程清清楚楚，管理者随时可以查看进度。

**不同的人看不同的界面。** 总控室值班员看全厂大屏，维修师傅看自己的工单列表，调度员看排产建议，管理层看能耗报表和节能数据。每个人只看到和自己工作相关的信息，不会眼花缭乱。

**一句话总结：** 智驭能效让钢铁厂从"靠经验管理"升级为"靠数据管理"，帮助工厂降低电费支出、减少设备故障停机时间、提高维修效率，最终实现降本增效和绿色制造。
