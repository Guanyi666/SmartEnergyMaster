## 一、电力负荷

### 电弧炉 EAF-01

```
P_rated = 40 MW（有功功率，代码常量）
stage_multiplier: STOPPED=0, IDLE=0.05, RUNNING=0.55, HIGH_LOAD=0.90

P_base(MW) = 40 × stage_multiplier
P_EAF = clamp(P_base + noise, 0, 50)
```

| 工况 | 功率范围（MW） | 15min 电量（kWh） | 日占比 |
|------|:---:|------|:---:|
| STOPPED | 0-0.1 | 0-25 | ~10% |
| IDLE | 1-3 | 250-750 | ~20% |
| RUNNING | 18-26 | 4500-6500 | ~50% |
| HIGH_LOAD | 30-42 | 7500-10500 | ~20% |

> RUNNING 功率 22 MW = 40×55%，是电弧炉平熔池操作典型工作点。5500 kWh/15min × 4 = 22 MWh/h，对应 ~55 t/h 产量 × 400 kWh/t ✓

### 钢包精炼炉 LF-01

```

```

### 连铸机 CC-01

```

```

### 循环水泵 PUMP-01

```

```

### 空压机 COMP-01

```

### 除尘系统 DC-01

```

## 五、碳排放

**当前因子**：0.010-0.020 tCO2/kWh（真实值的 ~20-35 倍）

**修正**：

```
CO2_grid    = usage_kwh × 0.0005703  tCO2    （中国电网 0.5703 kg/kWh）
CO2_process = usage_kwh × 0.00015    tCO2    （电极消耗 + 渣料分解）
CO2_EAF     = CO2_grid + CO2_process         （合计 ~0.00072 tCO2/kWh）
辅助设备     = usage_kwh × 0.0005703          （仅电网排放）
```

**示例**（EAF RUNNING, 22 MW × 0.25h = 5500 kWh）：

```
CO2 = 5500 × 0.0007203 = 3.96 tCO2/15min ≈ 380 tCO2/天
```

> 50 万吨钢厂年碳排放 15-25 万吨为合理范围 ✓

## 六、电价 — 陕西省大工业电价（1-10kV）

| 时段 | 时间范围 | 价格（元/kWh） |
|------|---------|:---:|
| **尖峰** | 7-8月 19:00-21:00、1/12月 18:00-20:00 | **1.25** |
| **峰** | 8:00-11:00、18:00-23:00（不含尖峰） | **0.95** |
| **平** | 6:00-8:00、11:00-18:00 | **0.60** |
| **谷** | 23:00-次日6:00 | **0.32** |
| **深谷** | 法定节假日 0:00-6:00 | **0.22** |

**判定逻辑**：

```python
def get_price_tier(nsm, month=None, is_holiday=False):
    hour = nsm // 3600
    if is_holiday and hour < 6: return "DEEP_VALLEY"
    if hour >= 23 or hour < 6:  return "VALLEY"
    if 6 <= hour < 8 or 11 <= hour < 18: return "FLAT"
    if month in (7,8) and 19 <= hour < 21: return "CRITICAL_PEAK"
    if month in (1,12) and 18 <= hour < 20: return "CRITICAL_PEAK"
    if 8 <= hour < 11 or 18 <= hour < 23: return "PEAK"
    return "FLAT"
```

**峰谷价差**：0.95 - 0.32 = 0.63 元/kWh（全国 0.5-0.9，合理）

## 七、设备运行率保证

### EAF 工况偏置

```python
def infer_stage_bias(row):
    usage = float(row["Usage_kWh"])
    hour = int(row["NSM"]) // 3600
    is_production = (8 <= hour < 22)
    
    if usage < 0.5 and not is_production: return "STOPPED"
    if usage < 5.0 or (usage < 15 and hour < 8): return "IDLE"
    if usage > 50: return "HIGH_LOAD"
    return "RUNNING"
```

| 工况 | 原占比 | 目标占比 |
|------|:---:|:---:|
| STOPPED | ~30% | **~10%** |
| IDLE | ~15% | **~20%** |
| RUNNING | ~35% | **~50%** |
| HIGH_LOAD | ~20% | **~20%** |

## 八、故障触发保证

后端检测规则 `SensorDataServiceImpl` 不变。数据泵故障注入与后端阈值对应：

| 故障 | 注入传感器值 | 后端阈值 | 触发 |
|------|-------------|---------|:---:|
| MECHANICAL_JAM | status→IDLE, vib→18-26 | vib>15 & status=1 | ✅ |
| COOLING_INTERRUPT | temp→1040-1165, press→30-60 | temp>1000 & press<50 | ✅ |
| INTERMITTENT_JAM | 间歇 vib→12.5-20.5 | vib>15 | ✅(间歇) |
| BEARING_WEAR | 渐进 vib↑ temp↑ | vib>15 | ✅(晚期) |

注入频率：1.2%/周期 × 3s = 平均 ~4 分钟/次故障事件

## 九、预测模型（LSTM 神经网络）

使用已有的 `ai_models/` LSTM 模型。功率放大后需要输入归一化：

```
sensor_data (0-15000) → PredictionClient 读取
    → Python prediction_service 归一化：÷100 → (0-150)
    → LSTM 推理 → 输出 (0-150)
    → 反归一化：×100 → 返回 (0-15000)
    → 前端展示预测曲线
```

LSTM 学的是时序变化模式，归一化后数据分布与训练时一致，无需重新训练。
