# 数据库结构重构方案

> **状态**：v4 方案（v3 经独立审查发现 5 个新问题：2 个 P0 + 2 个 P1 + 4 个 M，已修复）
> **日期**：2026-06-12
> **影响范围**：15 张表 → 17 张表，6 个 Entity 改动 + **3 个新增**（含 `MaintenancePersonnelArchive`），6+ 个 Service 改动

---

## Context（为什么改）

经过对 `deploy/init-sql/` 全部 10 个 SQL 文件、`backend/.../entity/` 11 个实体类、`backend/.../service/` 17 个 Service 实现的摸排，发现当前数据库结构存在以下**结构性问题**：

| 类别 | 问题 |
|------|------|
| **重复** | `maintenance_personnel` 与 `workorder_maintenance_personnel` 字段 90% 重复 |
| **死字段** | `spare_part_usage.created_at`（与 `used_at` 同值，VO 不暴露，无任何读/排序/过滤/前端展示）—— **仅此 1 个** |
| **JSON 滥用** | `maintenance_sop` 的 `steps / required_skills / required_tools / required_parts` 都是 JSON 字符串 |
| **孤儿外键** | `work_order.sop_id`、`repair_case.related_work_order_id` 无 FK 声明；`spare_part_usage.user_name` 字符串游离态 |
| **约束缺失** | `work_order.status`、`work_order.priority`、`device.status` 是自由字符串 |
| **索引不全** | `audit_log.actor_username`、`device.device_type`、`work_order.fault_type` 都没有索引 |
| **隐藏 P0** | `workorder_assignment.personnel_id` NOT NULL 与 FK ON DELETE SET NULL 冲突；`work_order_id` 缺 FK |

> **早期判断修正**（v2 评审发现）：`device.maintainer`、`work_order.assignee`、`work_order.latest_temperature/vibration/pressure` 均有完整数据流，**保留**。

**预期收益（v4 最终）：**
- 删除 1 个无用列（`spare_part_usage.created_at`）+ 3 个 `maintenance_personnel` 排班列
- 拆出 2 个子表（`maintenance_sop_step` + `maintenance_sop_required_part`）
- 给 4 个孤儿/缺失字段补 FK（`work_order.sop_id`、`repair_case.related_work_order_id`、`workorder_assignment.work_order_id`、修复 `workorder_assignment.personnel_id`）
- 新增 `spare_part_usage.user_id` FK（带 backfill）
- 5 个状态字段加 CHECK 约束
- 6+ 个字段加索引
- 新增 1 个 Entity（`MaintenancePersonnelArchive`）+ Mapper + FullVO

## ⚠️ v3 Self-Audit 关键发现（v3 评审发现）

v2 方案在实施前经独立探查（`seed_sop.sql`、`DataInitializer.java`、种子数据对比），发现 **8 个 CRITICAL 错误会导致迁移直接失败**：

| # | 改动组 | 错误 | 真实数据状态 | 修复策略 |
|---|--------|------|------------|----------|
| **A** | 1 | `user_id` NOT NULL 会失败 | E002-E006 的 employee_no 在 `sys_user` 中**无对应账号** | **不**强制 NOT NULL |
| **B** | 4 | `req->>'partCode'` 返回 NULL | `required_parts` 是**纯字符串数组** | 用 `jsonb_array_elements_text` 拿字符串 |
| **C** | 4 | `ON CONFLICT DO NOTHING` 缺 target | 新表无 UNIQUE 约束 | 去掉 ON CONFLICT |
| **D** | 4 | `step->>'no'`/`title` 全 NULL | `steps` 也是**纯字符串数组** | 用 `WITH ORDINALITY` 拿 step_no |
| **E** | 5 | 加 `work_order.sop_id` FK 失败 | 历史数据可能含孤立 FK | 先清孤儿 |
| **F** | 5 | 加 `repair_case.related_work_order_id` FK 失败 | DTO 允许**手填任意值** | 同上清孤儿 |
| **G** | 6 | CHECK 约束可能失败 | 历史数据需 DISTINCT 验证 | 用 `NOT VALID` + 异步校验 |
| **H** | 7 | `status='ACTIVE'` 覆盖已释放行 | `NOT NULL DEFAULT 'ACTIVE'` 误判 | 顺序：nullable → backfill → NOT NULL |

## 重构方案（7 大改动组，v4 最终版）

### 改动组 1：两张人员表明确定位（v4 修正）

**v4 修正**：
- A 错误：user_id 允许 NULL
- **P0-1**：新增 `MaintenancePersonnelArchive` Entity + Mapper
- **M-1**：必须先 `DROP CONSTRAINT ck_maintenance_personnel_workload` 再删列
- **M-2**：v3 漏写从 `maintenance_personnel` 删列的 SQL，v4 补全

**目标分工：**
| 表 | 定位 | 关键字段 |
|----|------|----------|
| `maintenance_personnel` | **员工档案**（人事系统对接） | `user_id` (FK sys_user), `employee_no`, `name`, `phone`, `email`, `specializations` (JSONB), `skill_level`, `certification` |
| `workorder_maintenance_personnel` | **维修排班业务档案** | `user_id` (FK sys_user, **可为 NULL**), `employee_no`, `avatar_color`, `current_workload`, `max_workload`, `is_on_duty` |

**关联方式：** 两张表各自 `user_id` 关联到 `sys_user`，**互不直接引用**。`maintenance_personnel.user_id` 用现有 FK（允许 NULL）。`workorder_maintenance_personnel.user_id` 新加 FK，**ON DELETE RESTRICT**，**允许 NULL**（兼容 E002-E006）。

**去重字段：**
- 从 `workorder_maintenance_personnel` 移除：`name / phone / email / specializations / skill_level / certification`
- 从 `maintenance_personnel` 移除：`current_workload / max_workload / is_on_duty`

**数据迁移：**
```sql
BEGIN;

-- 1. 备份
CREATE TABLE IF NOT EXISTS _backup_workorder_maintenance_personnel AS 
  SELECT * FROM workorder_maintenance_personnel;
CREATE TABLE IF NOT EXISTS _backup_maintenance_personnel AS 
  SELECT * FROM maintenance_personnel;

-- 2. 把 workorder_maintenance_personnel 的冗余字段搬到 maintenance_personnel
INSERT INTO maintenance_personnel (user_id, employee_no, name, phone, email, specializations, skill_level, certification)
SELECT NULL, wmp.employee_no, wmp.name, wmp.phone, wmp.email,
       wmp.specializations::jsonb, wmp.skill_level, wmp.certification
FROM workorder_maintenance_personnel wmp
ON CONFLICT (employee_no) DO NOTHING;

-- 3. 关联 sys_user（用 employee_no 当登录名匹配）—— 只有 E001 能匹配
UPDATE maintenance_personnel mp
SET user_id = su.id
FROM sys_user su
WHERE su.username = mp.employee_no AND mp.user_id IS NULL;

-- 4. 同步 user_id 到 workorder_maintenance_personnel
UPDATE workorder_maintenance_personnel wmp
SET user_id = mp.user_id
FROM maintenance_personnel mp
WHERE mp.employee_no = wmp.employee_no AND wmp.user_id IS NULL;

-- 5. M-1 修复：先 DROP 引用 current_workload/max_workload 的 CHECK 约束
ALTER TABLE maintenance_personnel DROP CONSTRAINT IF EXISTS ck_maintenance_personnel_workload;

-- 6. M-2 修复：从 maintenance_personnel 删排班列
ALTER TABLE maintenance_personnel 
  DROP COLUMN IF EXISTS current_workload,
  DROP COLUMN IF EXISTS max_workload,
  DROP COLUMN IF EXISTS is_on_duty;

-- 7. 删 workorder_maintenance_personnel 的冗余列
ALTER TABLE workorder_maintenance_personnel 
  DROP COLUMN name, DROP COLUMN phone, DROP COLUMN email, 
  DROP COLUMN specializations, DROP COLUMN skill_level, DROP COLUMN certification;

-- 8. 加 FK（ON DELETE RESTRICT，不强制 NOT NULL）
ALTER TABLE workorder_maintenance_personnel 
  ADD CONSTRAINT fk_wmp_user 
  FOREIGN KEY (user_id) REFERENCES sys_user(id) ON DELETE RESTRICT NOT VALID;
ALTER TABLE workorder_maintenance_personnel VALIDATE CONSTRAINT fk_wmp_user;

-- 9. 加 UNIQUE 索引（仅约束非 NULL 值）
CREATE UNIQUE INDEX IF NOT EXISTS ux_workorder_personnel_user 
  ON workorder_maintenance_personnel(user_id);

COMMIT;
```

### 改动组 4：拆分 `maintenance_sop` 的 JSON 数组

**v3 关键修正**：实际 JSON 是**纯字符串数组**，不是带 `partCode`/`name`/`quantity` 的对象数组。

#### 4.1 `maintenance_sop_step` —— 步骤子表

```sql
BEGIN;

CREATE TABLE IF NOT EXISTS maintenance_sop_step (
    id BIGSERIAL PRIMARY KEY,
    sop_id BIGINT NOT NULL REFERENCES maintenance_sop(id) ON DELETE CASCADE,
    step_no INTEGER NOT NULL,
    title VARCHAR(255) NOT NULL,
    content TEXT NOT NULL,
    expected_minutes INTEGER,
    warning TEXT,
    created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    UNIQUE (sop_id, step_no)
);
CREATE INDEX IF NOT EXISTS ix_sop_step_sop ON maintenance_sop_step(sop_id, step_no);

-- 数据迁移：steps 是纯字符串数组
INSERT INTO maintenance_sop_step (sop_id, step_no, title, content)
SELECT 
  s.id,
  t.ord::int,
  '步骤 ' || t.ord,
  t.step_text
FROM maintenance_sop s,
LATERAL jsonb_array_elements_text(s.steps::jsonb) WITH ORDINALITY AS t(step_text, ord)
WHERE s.steps IS NOT NULL AND s.steps::text != '[]';

ALTER TABLE maintenance_sop DROP COLUMN IF EXISTS steps;

COMMIT;
```

#### 4.2 `maintenance_sop_required_part` —— 所需备件子表

```sql
BEGIN;

CREATE TABLE IF NOT EXISTS maintenance_sop_required_part (
    id BIGSERIAL PRIMARY KEY,
    sop_id BIGINT NOT NULL REFERENCES maintenance_sop(id) ON DELETE CASCADE,
    part_id BIGINT REFERENCES spare_part(id) ON DELETE SET NULL,
    part_code VARCHAR(64) NOT NULL,
    quantity INTEGER NOT NULL DEFAULT 1,
    is_mandatory BOOLEAN NOT NULL DEFAULT TRUE,
    note VARCHAR(255),
    created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    CONSTRAINT ck_sop_req_part_qty CHECK (quantity > 0)
);
CREATE INDEX IF NOT EXISTS ix_sop_req_part_sop ON maintenance_sop_required_part(sop_id);
CREATE INDEX IF NOT EXISTS ix_sop_req_part_part ON maintenance_sop_required_part(part_id);

-- 数据迁移：required_parts 是纯字符串数组
INSERT INTO maintenance_sop_required_part (sop_id, part_id, part_code)
SELECT 
  s.id,
  sp.id,
  t.part_code
FROM maintenance_sop s,
LATERAL jsonb_array_elements_text(s.required_parts::jsonb) AS t(part_code)
LEFT JOIN spare_part sp ON sp.part_code = t.part_code
WHERE s.required_parts IS NOT NULL AND s.required_parts::text != '[]';

ALTER TABLE maintenance_sop DROP COLUMN IF EXISTS required_parts;

COMMIT;
```

### 改动组 5：补齐孤儿外键 + 索引（v4 加 P0-2 / P1-4 修复）

**v4 修正**：
- E/F 错误：先清孤儿
- **P0-2**：修复 `workorder_assignment.personnel_id` NOT NULL 与 `ON DELETE SET NULL` 冲突
- **P1-4**：补 `workorder_assignment.work_order_id` 的 FK
- K 错误：补新 FK 索引

```sql
BEGIN;

-- 1. 清孤儿（E 修复）：work_order.sop_id
UPDATE work_order SET sop_id = NULL 
WHERE sop_id IS NOT NULL 
  AND sop_id NOT IN (SELECT id FROM maintenance_sop);

-- 2. 清孤儿（F 修复）：repair_case.related_work_order_id
UPDATE repair_case SET related_work_order_id = NULL 
WHERE related_work_order_id IS NOT NULL 
  AND related_work_order_id NOT IN (SELECT id FROM work_order);

-- 3. P0-2 修复：workorder_assignment.personnel_id 改为可空
ALTER TABLE workorder_assignment ALTER COLUMN personnel_id DROP NOT NULL;

-- 4. P1-4 修复：workorder_assignment.work_order_id 加 FK
ALTER TABLE workorder_assignment 
  ADD CONSTRAINT fk_assignment_work_order 
  FOREIGN KEY (work_order_id) REFERENCES work_order(id) ON DELETE CASCADE NOT VALID;
ALTER TABLE workorder_assignment VALIDATE CONSTRAINT fk_assignment_work_order;

-- 5. 加 work_order.sop_id FK
ALTER TABLE work_order 
  ADD CONSTRAINT fk_work_order_sop 
  FOREIGN KEY (sop_id) REFERENCES maintenance_sop(id) ON DELETE SET NULL NOT VALID;
ALTER TABLE work_order VALIDATE CONSTRAINT fk_work_order_sop;

-- 6. 加 repair_case.related_work_order_id FK
ALTER TABLE repair_case 
  ADD CONSTRAINT fk_repair_case_work_order 
  FOREIGN KEY (related_work_order_id) REFERENCES work_order(id) ON DELETE SET NULL NOT VALID;
ALTER TABLE repair_case VALIDATE CONSTRAINT fk_repair_case_work_order;

-- 7. 加新 FK 索引（K 修复）
CREATE INDEX IF NOT EXISTS ix_work_order_sop_id ON work_order(sop_id);
CREATE INDEX IF NOT EXISTS ix_repair_case_related_work_order_id ON repair_case(related_work_order_id);
CREATE INDEX IF NOT EXISTS ix_workorder_assignment_work_order_id ON workorder_assignment(work_order_id);

-- 8. audit_log 新增字段
ALTER TABLE audit_log ADD COLUMN IF NOT EXISTS result VARCHAR(16);
ALTER TABLE audit_log ADD COLUMN IF NOT EXISTS duration_ms INT;

-- 9. CHECK 约束 result
ALTER TABLE audit_log ADD CONSTRAINT ck_audit_log_result 
  CHECK (result IS NULL OR result IN ('SUCCESS', 'FAILED'));

-- 10. audit_log 6 预留字段加 COMMENT（P1-3 决策）
COMMENT ON COLUMN audit_log.actor_user_id IS '预留：审计追溯，后续从 actor_username 解析 sys_user.id';
COMMENT ON COLUMN audit_log.request_id   IS '预留：request 拦截器写入，关联同一次 HTTP 调用';
COMMENT ON COLUMN audit_log.ip_address   IS '预留：request 拦截器写入，HttpServletRequest.getRemoteAddr()';
COMMENT ON COLUMN audit_log.user_agent   IS '预留：request 拦截器写入，HttpServletRequest.getHeader("User-Agent")';
COMMENT ON COLUMN audit_log.old_value    IS '预留：diff tracking，UPDATE 操作前的值';
```

### 改动组 7：`workorder_assignment` 释放语义优化

```sql
BEGIN;

-- 1. 加列（nullable，不设默认）
ALTER TABLE workorder_assignment ADD COLUMN IF NOT EXISTS status VARCHAR(16);

-- 2. backfill
UPDATE workorder_assignment SET status = 'RELEASED' WHERE released_at IS NOT NULL;
UPDATE workorder_assignment SET status = 'ACTIVE'   WHERE released_at IS NULL;

-- 3. 加 CHECK
ALTER TABLE workorder_assignment ADD CONSTRAINT ck_assignment_status 
  CHECK (status IN ('ACTIVE', 'RELEASED', 'TRANSFERRED'));

-- 4. SET NOT NULL
ALTER TABLE workorder_assignment ALTER COLUMN status SET NOT NULL;

-- 5. 设默认值
ALTER TABLE workorder_assignment ALTER COLUMN status SET DEFAULT 'ACTIVE';

COMMIT;
```
