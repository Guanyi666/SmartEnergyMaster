## 待办列表使用说明

- **优先级执行顺序**：P0 → P1 → P2 → P3，高优先级未完成前不启动低优先级待办项；
- **依赖关系遵循**：先完成依赖的待办项，再启动当前待办项，避免流程阻塞；
- **迭代规划参考**：按3轮迭代分配，每轮故事点总和控制在45-65之间，确保开发容量匹配；
- **验收标准执行**：开发完成后按用户故事中的验收标准逐项验证，全部满足视为"完成"；
- **版本管理**：后续需求变更需同步更新本列表。
