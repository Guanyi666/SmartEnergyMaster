## 前言

### 编写目的

本文档旨在明确智驭能效（SmartEnergyMaster）项目全生命周期的技术栈、工具链及 AI 辅助开发方案，通过统一技术选型标准，保障项目开发的一致性、高效性与稳定性，为系统研发、测试、部署及运维提供技术依据，支撑项目达成"数字孪生→趋势预测→智能告警→优化调度"全链路闭环的核心目标。

### 适用范围

适用于本项目全生命周期技术实施场景（含需求分析、架构设计、编码开发、测试验证、部署运维阶段），覆盖研发、测试、运维、技术管理等相关团队，作为技术落地、工具配置、跨团队协同的执行依据。

### 技术选型原则

- **一致性**：统一技术栈版本，保障开发、测试、生产环境的兼容性；
- **适配性**：聚焦 PC 端大屏监控、移动端轻量查看等工业场景；
- **高效性**：选用成熟工具链 + AI 辅助编程（LLM诊断、Agent协作），降低开发成本；
- **稳定性**：优先选用企业级成熟技术组件（Spring Boot、TimescaleDB、Redis），保障生产环境运行可靠；
- **可扩展性**：保留功能扩展、系统集成的技术接口，支持后续业务迭代。

## 技术风险与应对措施

| 风险 | 等级 | 应对措施 |
|------|------|----------|
| LLM API 调用延迟 >30s | 中 | 设置超时降级、异步调用、预设模板兜底 |
| TimescaleDB 超表查询性能下降 | 低 | 合理建索引、使用连续聚合、数据压缩策略 |
| Redis 内存溢出 | 低 | 设置 maxmemory + 淘汰策略（allkeys-lru） |
| Python AI 服务与 Java 后端通信失败 | 中 | 健康检查 + 重试机制 + 降级策略 |
| Vue 3 响应式性能问题 | 低 | 虚拟滚动、懒加载、ECharts 按需引入 |
