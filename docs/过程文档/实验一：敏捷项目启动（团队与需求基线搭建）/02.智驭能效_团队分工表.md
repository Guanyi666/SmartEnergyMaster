## 二、各角色详细职责

### 2.1 产品负责人（PO）

| 职责领域 | 具体工作 |
|----------|----------|
| 需求管理 | 主导需求分析，梳理能源监控、设备管理、故障诊断、智能调度等用户故事，构建用户故事地图与产品待办列表 |
| 优先级决策 | 确定"传感器数据采集→实时监控大屏→告警与工单→AI诊断→智能调度"的需求优先级排序 |
| 验收决策 | 每轮迭代结束前验收交付成果，确认是否满足验收标准 |
| 沟通桥梁 | 对接指导老师，同步需求变更与反馈，向团队传递准确信息 |

### 2.2 敏捷教练（SM）

| 职责领域 | 具体工作 |
|----------|----------|
| 流程保障 | 引导团队严格执行迭代规划、每日站会、迭代评审、迭代回顾等 Scrum 会议 |
| 障碍清除 | 主动收集开发过程中的技术难点（AI模型集成、Redis配置等），协调资源协助解决 |
| 团队赋能 | 引导团队自组织运作，鼓励成员主动承担责任 |
| 文档统筹 | 督促各环节输出规范材料（燃尽图、站会记录、迭代报告），确保过程可追溯 |

### 2.3 开发团队（DT）

开发团队为跨职能自组织团队，8 名成员共同承担开发、测试、文档等任务：

| 职责领域 | 具体工作 |
|----------|----------|
| 任务执行 | 根据迭代待办列表，自组织拆分"后端API开发""前端页面开发""AI模型训练"等任务并落实 |
| 质量把控 | 严格遵守代码规范，编写单元测试，确保接口响应时间 ≤ 500ms、大屏刷新周期 ≤ 5s |
| 主动协作 | 每日站会同步个人进度、遇到的问题及计划，主动协助团队成员 |
| 参与改进 | 参与迭代回顾会议，提出流程优化建议 |
