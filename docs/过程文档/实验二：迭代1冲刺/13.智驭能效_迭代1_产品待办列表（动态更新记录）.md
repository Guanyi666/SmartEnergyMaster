## 关键决策说明

### UPD-001：Redis 基础设施提前

**背景**：原计划 Redis 在迭代2引入（配合缓存和限流），但开发中发现前端 DashboardView 5s 轮询 4 个 API，每次全量查 DB，3 台设备产生大量重复查询。

**决策**：将 Redis 缓存基础设施从迭代2提前到迭代1，涵盖：
- `CacheService` 读穿透缓存（device:latest、device:list、dashboard:summary）
- `@RateLimit` 登录防爆破限流
- `@DistributedLock` 传感器上报互斥锁
- `RedisConfig` 序列化配置

**影响**：增加 15 故事点，但大幅降低 DB 压力，且 Redis 在 docker-compose 中可快速启动。

### UPD-003：分析页增强降级

**背景**：分析页 Enhancement（时间选择器/多设备对比/电价着色）在迭代规划时标注 P2，但开发中发现 P1 任务量已饱和。

**决策**：仅保留基础历史曲线（S3.4.1），增强功能移至迭代2。
