## 缺陷管理概述

本文档记录迭代1冲刺期间（2026年6月10日—6月12日）发现的所有软件缺陷，包括缺陷描述、复现步骤、根因分析和修复验证全过程。缺陷按优先级分为三级：

- **🔴 高（阻塞性）**：导致核心功能不可用或数据丢失，必须立即修复
- **🟠 中（功能性）**：功能可用但存在缺陷，影响用户体验或数据准确性
- **🟢 低（体验/文档类）**：体验瑕疵或文档问题，不影响核心功能

**缺陷发现渠道**：
1. 全链路联调测试（6月12日上午集中联调）
2. 每日站会后手动冒烟测试
3. Code Review 过程中发现的代码缺陷
4. 前端与后端对接时的接口不一致问题

### BUG-003：登录失败后 rate limit key 未过期

| 项目 | 内容 |
|------|------|
| 缺陷编号 | BUG-003 |
| 优先级 | 🟠 中 |
| 发现日期 | 2026-06-09 |
| 发现人 | 彭锐豪 |
| 所属模块 | 限流（Epic 11） |
| 问题描述 | 登录限流 key (rate:login:{IP}) 在 60s 窗口过后未自动过期，导致持续返回 429 |
| 复现步骤 | 1. 连续 3 次错误密码登录<br>2. 第 4 次触发 429<br>3. 等待 65 秒<br>4. 再次尝试 → 仍返回 429 |
| 根因分析 | Lua 脚本中 PEXPIRE 的过期时间是 window * 1000（毫秒），但 Redis ZSET 的 TTL 在 ZADD 后未刷新，导致 key 永久存在 |
| 解决方案 | 在 Lua 脚本 ZADD 后再次调用 PEXPIRE 刷新 TTL（代码修正后每次 ZADD 都会 PEXPIRE） |
| 修复人 | 彭锐豪 |
| 修复日期 | 2026-06-09 |
| 验证人 | QA |
| 验证结果 | ✅ 验证通过，等待 65s 后限流解除 |

### BUG-006：MybatisPlusConfig Bean 名称冲突 → 后端无法启动

| 项目 | 内容 |
|------|------|
| 缺陷编号 | BUG-006 |
| 优先级 | 🔴 高（阻塞性） |
| 发现日期 | 2026-06-12 |
| 发现人 | 段冠屹 |
| 所属模块 | 后端基础设施 |
| 问题描述 | workorder 模块合并到 backend 后，com.smartenergy.workorder.config.MybatisPlusConfig 与 com.smartenergy.backend.config.MybatisPlusConfig 产生同名 Bean 冲突，Spring Boot 启动时抛出 ConflictingBeanDefinitionException，应用无法启动 |
| 复现步骤 | 1. 合并 workorder-backend 代码到 backend<br>2. 执行 mvn spring-boot:run<br>3. 观察启动日志 → 崩溃 |

| 解决方案 | 删除 workorder 的 MybatisPlusConfig，将其 setOverflow(false) + setMaxLimit(500) 合并到主配置中 |
| 修复人 | 段冠屹 |
| 修复日期 | 2026-06-12 |
| 验证人 | 段冠屹 |

### BUG-007：JWT 403 认证失败 → 所有 API 返回 403

| 项目 | 内容 |
|------|------|
| 缺陷编号 | BUG-007 |
| 优先级 | 🟠 中（功能性） |
| 发现日期 | 2026-06-12 |
| 发现人 | 段冠屹 |
| 所属模块 | 认证模块（Security） |
| 问题描述 | 登录成功后获取 Token 正常，但使用 Token 请求任何受保护 API 均返回 HTTP 403。经排查，@Value("${jwt.secret}") 在 OncePerRequestFilter 中注入不可靠，JwtAuthenticationFilter 使用空密钥验签导致 verify() 返回 false，请求未经认证到达 Security 链被拒绝 |
| 复现步骤 | 1. POST /api/auth/login → 成功获取 Token<br>2. GET /api/devices with Bearer Token → 403 |
| 根因分析 | @Value 注入在 OncePerRequestFilter 子类中不可靠，注入空字符串而非默认值；空密钥使 JWTUtil.verify() 返回 false（不抛异常），SecurityContext 未装配 |
| 解决方案 | 创建 JwtConfig（@ConfigurationProperties）统一管理密钥，JwtAuthenticationFilter 改为 @Autowired 注入；同时修复 expire_time 类型安全读取（Number → longValue） |
| 修复人 | 段冠屹 |
| 修复日期 | 2026-06-12 |
| 验证人 | 段冠屹 |
| 验证结果 | ✅ 验证通过，全部受保护 API 返回 200 |
