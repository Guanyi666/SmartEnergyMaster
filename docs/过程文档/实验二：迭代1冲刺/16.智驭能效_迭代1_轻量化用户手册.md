## 系统概述

智驭能效（SmartEnergyMaster）是一个面向钢铁厂的工业能源管理平台，用于监控核心设备（电弧炉、循环水泵、空压机）的实时运行状态，自动检测设备故障并生成维修工单，结合分时电价给出 AI 调度建议。

**迭代1 版本功能：**
- 用户登录与权限控制
- 设备台账管理
- 实时监控大屏（5 秒自动刷新）
- 设备故障自动检测与告警
- 维修工单全流程管理
- 历史数据趋势分析
- 分时电价调度建议

## 环境要求与快速启动

### 环境要求

- Docker Desktop
- Node.js ≥ 16
- Java ≥ 17
- Maven ≥ 3.6
- Python ≥ 3.10

### 启动步骤

```bash
# 步骤 1：启动数据库和 Redis
cd deploy
docker compose up -d

# 步骤 2：启动后端（新终端）
cd backend
mvn spring-boot:run

# 步骤 3：启动前端（新终端）
cd frontend
npm install   # 首次运行
npm run dev

# 步骤 4：启动数据仿真器（新终端，可选）
cd data
pip install -r requirements.txt   # 首次运行
python data_pump.py
```

## 功能操作指南

### 登录系统

1. 浏览器打开 http://localhost:5173 → 自动跳转到登录页
2. 输入内置管理员账号：`admin` / `123456`
3. 点击"登录" → 进入监控大屏

> **提示**：首次使用可点击"注册"创建新账号，新账号默认角色为操作员（OPERATOR）。

### 实时监控大屏（Dashboard）

大屏是系统核心操作界面，自动每 5 秒刷新一次数据。

**界面布局（从上到下）：**

| 区域 | 内容 | 操作 |
|------|------|------|
| 顶部 | 3 张指标卡：全厂总功率 / 累计碳排放 / 当前电价区间 | 只读 |
| 中部左 | 双仪表盘：温度（°C）/ 压力（kPa） | 切换焦点设备后自动更新 |
| 中部右 | AI 调度建议卡片 + 待处理工单列表 | 点击"确认处理"/"已修复"按钮 |
| 底部 | 设备状态磁贴条 | 点击磁贴切换监控目标 |

**切换监控设备**：
- 方式一：顶部下拉框选择设备
- 方式二：点击底部设备磁贴

**处理工单**：
- 在右侧"待处理维修工单"面板找到工单
- 点击"确认处理"：工单进入处理中状态，设备显示"维修中"
- 点击"已修复"：工单闭环，设备恢复正常状态

### 能耗分析页（Analysis）

1. 左侧导航栏点击"能耗分析"
2. 页面展示选中设备过去 24 小时的温度和功率趋势曲线
3. 使用顶部设备下拉框切换分析不同设备

### 设备管理页（Devices）

1. 左侧导航栏点击"设备管理"
2. 页面以表格展示所有注册设备

**新增设备**：
- 点击"添加设备"按钮 → 弹出表单
- 填写设备编码（唯一）、设备名称、设备类型、位置、维护人、描述
- 点击"确认"提交

**编辑设备**：
- 点击表格每行末尾的"编辑"按钮 → 弹出回填表单
- 修改后点击"确认"

**删除设备**：
- 点击"删除"按钮 → 二次确认弹窗 → 确认后删除

> ⚠️ 注意：设备编码全局唯一，重复提交会提示"设备编码已存在"。

### 数据仿真器（开发者/演示用）

仿真器模拟 3 台钢铁厂设备（电弧炉/水泵/空压机）的传感器数据推送：

```bash
cd data
python data_pump.py
```

启动后：
- 先回填 24 小时历史数据（96 个点，约 5 秒）
- 再进入实时推送模式（每 3 秒推送一次 3 台设备数据）
- 概率性注入故障（约 0.6%-0.9% 概率），用于测试告警功能

按 `Ctrl+C` 停止仿真器。

## 常见问题

### Q1：前端打不开或一直加载？

- 确认后端已启动（访问 http://localhost:8080/swagger-ui.html 验证）
- 确认前端已启动（终端显示 `VITE v6.x ready`）

### Q2：登录失败提示"用户名或密码错误"？

- 默认管理员账号：`admin` / `123456`
- 如果数据库刚初始化，等待几秒让 DataInitializer 创建默认用户

### Q3：大屏数据不刷新？

- 检查仿真器是否在运行（data_pump.py）
- 如果仿真器未运行，大屏数据将是最后一次上报的缓存数据
- 切换浏览器标签页后再切回来会立即触发刷新

### Q4：Swagger 页面中文乱码？

- 浏览器右键 → 编码 → 选择 UTF-8

### Q5：仿真器输出中文乱码（Windows CMD）？

- 推荐在 Git Bash、VS Code 终端或 PyCharm 中运行（UTF-8 环境）
- 或在 data_pump.py 开头加 `# -*- coding: utf-8 -*-`

### Q6：Docker 启动失败？

```bash
# 检查 Docker 是否运行
docker ps

# 查看容器日志
docker logs smart_energy_db

# 重启容器
docker compose down && docker compose up -d
```

## 常见问题排查（FAQ）

### Q1：前端页面显示空白/白屏？
**原因**：后端未启动或端口冲突  
**排查**：
1. 检查后端是否启动：浏览器访问 `http://localhost:8080/swagger-ui.html`
2. 如无法访问，执行 `cd backend && mvn spring-boot:run`
3. 如端口 8080 被占用，杀进程后重启

### Q2：大屏仪表盘数据不刷新？
**原因**：仿真器未启动或网络断开  
**排查**：
1. 检查仿真器终端是否有"POST /api/sensor/upload → 200"日志
2. 如仿真器静默退出，重新执行 `cd data && python data_pump.py`
3. 确认仿真器和后端在同一网络

### Q3：登录后跳转到空白页？
**原因**：浏览器 localStorage 残留旧 Token  
**解决**：F12 → Application → Local Storage → 清除 `smart-energy-token` → 刷新页面重新登录

### Q4：设备状态一直显示 OFFLINE？
**原因**：仿真器未在推送数据，或设备编码不匹配  
**排查**：
1. 检查仿真器日志中 deviceCode 是否与数据库中一致
2. 调用 GET `/api/sensor/latest/EAF-01` 确认最新数据时间是否在 5 分钟内

### Q5：Swagger UI 中文乱码？
**原因**：浏览器编码设置  
**解决**：Chrome → 设置 → 编码 → UTF-8；或在地址栏强制刷新 Ctrl+F5

### Q6：docker-compose up 失败？
**原因**：端口 5432（TimescaleDB）或 6379（Redis）被占用  
**排查**：
1. `netstat -ano | findstr 5432` 查看是否有其他 PG 实例
2. `netstat -ano | findstr 6379` 查看是否有其他 Redis 实例
3. 停止冲突服务后重试 `docker-compose up -d`

### Q7：仿真器 Windows CMD 中文乱码？
**解决**：推荐在 Git Bash 或 IDE 终端中运行（默认 UTF-8 环境）。CMD 的 GBK 编码与 Python UTF-8 输出不兼容。

### Q8：工单"确认处理"后设备状态未变？
**原因**：可能是旧版本 Bug（已修复）  
**当前版本**：PATCH /api/work-orders/{id}/status 会自动联动 DeviceService.updateDeviceStatus。如仍不更新，检查后端日志是否有 ERROR。

## 系统配置参考

### application.yml 关键配置

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| server.port | 8080 | 后端端口 |
| spring.datasource.url | jdbc:postgresql://localhost:5432/smart_energy | DB 连接 |
| spring.data.redis.host | localhost | Redis 地址 |
| jwt.secret | SmartEnergyMasterSecretKey | JWT 密钥（生产需更换） |
| jwt.expiration-ms | 86400000 | Token 有效期（24h） |
| prediction.service.base-url | http://localhost:8000 | AI 预测服务地址 |

### 仿真器配置（data_pump.py）

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 推送间隔 | 3s | 每台设备传感器数据推送间隔 |
| 历史回填 | 24h / 96点 | 15min 间隔的历史数据量 |
| 故障概率 | ~5% / 次 | 每次推送随机注入故障的概率 |
| 后端地址 | http://localhost:8080 | API 端点 |

### Docker 端口映射

| 服务 | 容器端口 | 主机端口 | 
|------|---------|---------|
| TimescaleDB | 5432 | 5432 |
| Redis | 6379 | 6379 |
| 后端 | 8080 | 8080 |
| 前端 (Vite) | 5173 | 5173 |
| AI 预测服务 | 8000 | 8000 |
