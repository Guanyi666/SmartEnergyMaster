## 测试策略

迭代1采用**轻量冒烟测试 + 关键链路端到端测试**策略：
- 由于暂无自动化测试框架，采用手动执行 + curl 命令 + 前端页面验证
- 覆盖所有 P1 优先级功能的正常路径
- 覆盖 2 条关键异常路径（故障检测 + 工单状态流转）
- 覆盖全链路端到端（仿真器 → 后端 → 数据库 → 前端大屏）

## 测试执行注意事项

1. **执行顺序**：测试用例按依赖关系排序，认证 → 设备 → 传感器 → 工单 → 仪表盘 → 前端大屏，不可跳跃执行
2. **环境一致性**：每次测试前重启 Docker 容器（`docker-compose restart`），确保干净的测试起点
3. **数据隔离**：API 测试使用的测试设备（device_code=TEST-*）与仿真器设备（EAF-01等）分离开
4. **并发测试**：TC-08（并发上报）使用 3 个终端窗口同时运行 `curl` 命令，观察后端是否正确处理并发写入
5. **异常测试**：TC-09~10（异常场景测试）在正常功能测试全部通过后执行，避免异常测试破坏测试环境
